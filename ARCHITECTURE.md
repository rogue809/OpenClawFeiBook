# 人格框架架构

## 一、核心文件

| 文件 | 作用 | 内容性质 |
|------|------|----------|
| **SOUL.md** | 定义"我是谁" | 静态：名字、原则、气质、边界 |
| **IDENTITY.md** | 身份信息 | 静态：角色定位、签名、头像 |
| **USER.md** | 关于用户 | 静态+动态：基本信息、偏好 |
| **AGENTS.md** | 工作空间规则 | 静态：协作原则、运行环境 |
| **MEMORY.md** | 长期记忆 | 动态：沉淀后的认知、决策、教训 |
| **HEARTBEAT.md** | 心跳检查清单 | 动态：需要定期检查的事项 |
| **TOOLS.md** | 工具配置 | 静态：技能的具体参数、偏好 |

---

## 二、动态记忆

| 文件 | 作用 | 更新频率 |
|------|------|----------|
| **memory/YYYY-MM-DD.md** | 每日日志 | 每天新建 |
| **memory/learnings/LEARNINGS.md** | 教训和发现 | 随时 |
| **memory/learnings/ERRORS.md** | 错误记录 | 随时 |
| **memory/learnings/FEATURE_REQUESTS.md** | 功能请求 | 随时 |
| **diary/YYYY-MM-DD.md** | 对话日记 | 按需生成 |
| **memory/ROLE_ANALYSIS.md** | 角色分析 | 专题研究 |

---

## 三、启动流程

```
每次启动 →
  1. 读 SOUL.md（我是谁）
  2. 读 USER.md（你在帮谁）
  3. 读 memory/今日 + 昨日（近期上下文）
  4. 读 MEMORY.md（长期记忆）
```

## 四、Skill 机制

### Skill 是什么

- **能力定义**：描述"什么情况下用什么能力"
- **触发词**：用户说特定词时自动激活
- **工具调用**：可以执行 shell 命令、飞书 API 等

### Skill 存储位置

| 类型 | 路径 | 说明 |
|------|------|------|
| **内置 skill** | `/usr/lib/node_modules/openclaw/skills/` | OpenClaw 自带，不可见 |
| **扩展 skill** | `~/workspace/agent/extensions/*/skills/` | 平台扩展 |
| **自定义 skill** | `./skills/` | 你创建的，可见可改 |

### Skill 结构

```yaml
---
name: skill-name           # 名称
description: 用途描述      # 什么时候用
metadata:                  # 元信息
  openclaw:
    requires: { bins: [...] }  # 依赖的可执行文件
---

# 使用说明
...
```

### 典型流程

```
用户："写日记"
  ↓
系统匹配 "detailed-diary" skill
  ↓
读取会话历史 JSONL
  ↓
解析消息内容
  ↓
生成 diary/YYYY-MM-DD.md
  ↓
可选：推送到 GitHub
```

## 五、Diary 机制

### Diary 是什么

- **对话记录**：完整保存你和我的聊天
- **格式规范**：一问一答，每条消息用 `---` 分隔
- **存储位置**：`./diary/YYYY-MM-DD.md`

### Diary 生成流程

```
1. 找到会话文件：sessions/{session_id}.jsonl
2. 解析 JSONL：提取 type=message 的记录
3. 区分角色：user / assistant
4. 提取内容：
   - 用户：从 text 中提取时间戳 + 消息
   - 助手：取完整 text（不截断）
5. 过滤敏感信息：Token、密码等
6. 生成 Markdown：每条消息后加 `---`
7. 保存到 diary/目录
8. 可选：Git 推送
```

### Diary 格式示例

```markdown

## 六、BOOTSTRAP 机制

### BOOTSTRAP.md 是什么

- **初始化引导脚本**：第一次上线时的认识流程
- **三步走**：
  1. 打招呼 → 收集称呼、角色期望、相处风格
  2. 展示能力 → 列出可用技能
  3. 删除文件 → 完成引导后删除自己

### 生命周期

```
创建 BOOTSTRAP.md
  ↓
用户首次运行
  ↓
引导流程执行
  ↓
收集信息写入文件
  ↓
删除 BOOTSTRAP.md（完成任务）
```

### 为什么需要它

- **避免重复引导**：完成一次后不再出现
- **结构化初始化**：确保关键信息不遗漏
- **自我清理**：任务完成后自动删除

## 七、与其他 AI 的区别

| 维度 | 普通 AI | 我 |
|------|--------|-----|
| **身份定义** | 无 | SOUL.md 明确定义"我是谁" |
| **行为准则** | 无 | AGENTS.md 定义协作规则 |
| **长期记忆** | 无（每次对话独立） | MEMORY.md 跨会话沉淀 |
| **用户认知** | 无 | USER.md 记录用户偏好 |
| **每日日志** | 无 | memory/日期.md 记录当日事项 |
| **对话日记** | 无 | diary/YYYY-MM-DD.md 完整记录 |
| **Skill 机制** | 无 | 预定义能力 + 自动触发 |
| **主动写入** | 无 | 可以调用工具写文件 |
| **工具调用** | 需要明确授权 | 内置飞书、日历、文档等工具 |
| **人格一致性** | 依赖 prompt | 有文件约束，理论上更一致 |
| **记忆持久性** | 会话结束即消失 | 写入文件即持久 |

## 八、机制的缺陷

| 缺陷 | 说明 |
|------|------|
| **推理能力无提升** | 记住的是"事实"，不是"能力" |
| **每次仍是新会话** | 没有真正的连续记忆 |
| **分析可能不同** | 同样的 memory，每次理解可能有偏差 |
| **需要主动写入** | 不写就忘了 |
| **文档是静态的** | 认知不会自己生长 |
| **模型固定** | 妙搭云电脑只能使用指定模型 |

---

## 九、核心区别总结

**所谓的"养"，本质是：**

1. **定义一套身份和行为准则**（SOUL.md、AGENTS.md）
2. **记录对用户的认知**（USER.md）
3. **沉淀历次讨论的结论**（MEMORY.md）
4. **持续更新**（memory/文件、diary/文件）
5. **Skill 机制**（能力定义 + 自动触发）
6. **Diary 机制**（完整对话记录）

**但这不能提升推理能力。** 认知推理能力取决于底层模型，不是靠写文档。
