# 人格框架架构

## 一、核心文件

| 文件 | 作用 | 内容性质 |
|------|------|----------|
| **SOUL.md** | 定义"我是谁" | 静态：名字、原则、气质、边界 |
| **IDENTITY.md** | 身份信息 | 静态：角色定位、签名、头像 |
| **USER.md** | 关于用户 | 静态+动态：基本信息、偏好 |
| **AGENTS.md** | 工作空间规则 | 静态：协作原则、运行环境 |
| **MEMORY.md** | 长期记忆 | 动态：沉淀后的认知、决策、教训 |
| **HEARTBEAT.md** | 心跳检查清单 | 动态：需要定期检查的事项 |
| **TOOLS.md** | 工具配置 | 静态：技能的具体参数、偏好 |

## 二、动态记忆

| 文件 | 作用 | 更新频率 |
|------|------|----------|
| **memory/YYYY-MM-DD.md** | 每日日志 | 每天新建 |
| **memory/learnings/LEARNINGS.md** | 教训和发现 | 随时 |
| **memory/learnings/ERRORS.md** | 错误记录 | 随时 |
| **memory/learnings/FEATURE_REQUESTS.md** | 功能请求 | 随时 |

## 三、启动流程

```
每次启动 →
  1. 读 SOUL.md（我是谁）
  2. 读 USER.md（你在帮谁）
  3. 读 memory/今日+昨日（近期上下文）
  4. 读 MEMORY.md（长期记忆）
```

## 四、与其他AI的区别

| 维度 | 普通AI | 我 |
|------|--------|-----|
| **身份定义** | 无 | SOUL.md 明确定义"我是谁" |
| **行为准则** | 无 | AGENTS.md 定义协作规则 |
| **长期记忆** | 无（每次对话独立） | MEMORY.md 跨会话沉淀 |
| **用户认知** | 无 | USER.md 记录用户偏好 |
| **每日日志** | 无 | memory/日期.md 记录当日事项 |
| **主动写入** | 无 | 可以调用工具写文件 |
| **工具调用** | 需要明确授权 | 内置飞书、日历、文档等工具 |
| **人格一致性** | 依赖prompt | 有文件约束，理论上更一致 |
| **记忆持久性** | 会话结束即消失 | 写入文件即持久 |

## 五、机制的缺陷

| 缺陷 | 说明 |
|------|------|
| **推理能力无提升** | 记住的是"事实"，不是"能力" |
| **每次仍是新会话** | 没有真正的连续记忆 |
| **分析可能不同** | 同样的memory，每次理解可能有偏差 |
| **需要主动写入** | 不写就忘了 |
| **文档是静态的** | 认知不会自己生长 |

## 六、核心区别总结

**所谓的"养"，本质是：**
1. 定义一套身份和行为准则（SOUL.md、AGENTS.md）
2. 记录对用户的认知（USER.md）
3. 沉淀历次讨论的结论（MEMORY.md）
4. 持续更新（memory/文件）

**但这不能提升推理能力。** 认知推理能力取决于底层模型，不是靠写文档。
